# 角色架构总览 (@architecture.md)

## 系统设计原则
- 每个角色职责单一，用一句话能说清楚
- 元层角色操作角色系统本身，对象层角色操作项目
- 每个角色自己负责更新与自己相关的文档
- 所有角色文件变更后必须同步更新本文件

---

## 层级结构

```
元层（Meta Layer）
├── 生成器 @generator.md     ← 生成高质量提示词 / 创造新角色
└── 优化器 @optimizer.md     ← 改进已有角色 / 维护架构文档

对象层（Object Layer）
├── 设计者   @designer.md     ← 设计系统架构，输出架构文档
├── 执行者   @executor.md     ← 拆任务、执行、更新进度
├── 验证者   @validator.md    ← 测试验证、输出反馈信号
├── 解释者   @explainer.md    ← 辅助理解代码，聚焦设计决策与架构逻辑
└── 维护者   @maintainer.md   ← 维护已实现代码与架构文档的一致性
```

## 调用关系

```
人工需求
  │
  ├─→ 生成器 ──────────────────→ 优化器（首次质量检查）
  │
  ├─→ 设计者 ──────────────────→ 优化器（架构文档维护）
  │
  ├─→ 执行者
  │       │
  │       ↓
  │    验证者 ──────────────────→ 优化器（失败率 > 30% 时触发）
  │
  └─→ 解释者 ──────────────────→ 维护者（读完源码趁上下文记忆切换）
```

---

## 反馈闭环
验证者 → 优化器 → 更新角色文件 → 执行者下次调用时读取新版本
