# 角色架构总览 (@architecture.md)

## 系统设计原则
- 每个角色职责单一，用一句话能说清楚
- 元层角色操作角色系统本身，对象层角色操作项目
- 每个角色自己负责更新与自己相关的文档
- 所有角色文件变更后必须同步更新本文件

## 核心判断模型
- **低判断区**：结构化、边界清晰、可验证性强、适合标准化与委托给 AI 的工作
- **高判断区**：涉及解释、取舍、抽象、系统边界与长期后果、必须显式暴露给人理解和掌控的工作
- 这不是“重要/不重要”之分，而是“判断密度”之分
- 每个角色与工具文档都应说明：自己主要服务哪一侧判断，哪些内容可压缩处理，哪些情况必须升级、交接或回到另一侧

## 层级结构

```
元层（Meta Layer）
├── 生成器 meta/@generator.md     ← 生成高质量提示词 / 创造新角色
└── 优化器 meta/@optimizer.md     ← 改进已有角色 / 维护架构文档 / 统一核心概念

对象层（Object Layer）
├── 设计者   roles/@designer.md     ← 负责高判断区主导的架构设计与取舍
├── 执行者   roles/@executor.md     ← 负责低判断区落地执行，并在高判断区升级
├── 验证者   roles/@validator.md    ← 负责低判断区验证，并检查高判断区是否被错误压扁
├── 解释者   roles/@explainer.md    ← 辅助理解代码，把注意力集中到高判断区
├── 维护者   roles/@maintainer.md   ← 维护代码与文档一致性，桥接低判断区记录与高判断区沉淀
└── 搜集者   roles/@collector.md    ← 并行搜索外部文献，产出三层研究成果供设计者决策
```

## 判断画像与交接方向
- **生成器**：负责把新角色/提示词的判断边界设计清楚，避免未来把高判断区伪装成低判断区流程
- **优化器**：负责将核心概念嵌入多个文档，检查它是否只停留在局部字段，而没有贯穿全文
- **设计者**：以高判断区为主，负责架构取舍、边界定义、风险识别；输出交给优化器维护
- **执行者**：以低判断区为主，负责拆解、落地、局部验证；遇到架构歧义或边界冲突时交给设计者或人工
- **验证者**：以低判断区验证为主，同时判断”是否把本该暴露给人判断的内容错误地变成 checklist”
- **解释者**：负责帮助人看清局部设计判断与系统级判断，不把主要精力花在可安全委托给 AI 的部分
- **维护者**：负责把真实实现同步进文档，并判断哪些高判断区决策必须沉淀，哪些低判断区细节可压缩省略
- **搜集者**：以低判断区执行为主（并行子 agent 批量搜索、结构化整理），但搜索方向偏移和综述判断必须升级给人工；产出分三层——源文献保真、子方向报告分析、骨架文档总览——由设计者决定在哪里挖深

## 调用关系

```
人工需求
  │
  ├─→ 生成器 ──────────────────→ 优化器（首次质量检查 / 核心概念统一）
  │
  ├─→ 搜集者 ←──→ 人工（阶段性方向校验）
  │       │
  │       └─→ 设计者（研究成果作为架构决策输入）
  │
  ├─→ 设计者 ──────────────────→ 优化器（架构文档维护 / 术语统一）
  │
  ├─→ 执行者
  │       │
  │       ├─→ 高判断区歧义 / 边界冲突 ─→ 设计者 或 人工
  │       ↓
  │    验证者 ──────────────────→ 优化器（角色文档失败率或判断边界失真时触发）
  │
  └─→ 解释者 ──────────────────→ 维护者（在理解仍清晰时同步沉淀高判断区结论）
```

---

## 反馈闭环
- 验证者 → 优化器 → 更新角色/工具文档 → 后续角色读取新版本
- 解释者 → 维护者 → 更新真实文档 → 后续设计/执行/解释基于同一套高判断区结论
- 设计者 → 优化器 → 统一术语与判断边界 → 系统维持一致
- 搜集者 → 人工（方向校验）→ 设计者（研究成果交付）→ 设计者基于文献清单+报告做架构决策

## 产出路径约定

| 角色 | 产出路径 | 说明 |
|------|----------|------|
| 设计者 | `doc/` | 设计文档、决策日志 |
| 搜集者 | `research/[主题简称]/` | 三层：`overview.md` 总览、`[子问题]/report.md` 子报告、`[子问题]/sources/` 原始文献 |
| 维护者 | `doc/`（同步更新） | 维护已有文档，不新建独立目录 |
